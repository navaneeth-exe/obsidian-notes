## **Definition**

A page table is a data structure used to **map virtual page numbers (VPN) to physical page numbers (PPN)** for address translation.

---

## **Basic Concept**

- Each program has its **own page table**
- Stored in **main memory**
- Used by CPU to convert **virtual address → physical address**

---

## **Structure of Page Table**

- Contains **one entry for each virtual page**
- Each entry includes:
    - **Physical Page Number (PPN)**
    - **Valid Bit**

---

## **Valid Bit**

- **1 → Page is present in main memory**
- **0 → Page not present (page fault occurs)**

---

# **Address Translation using Page Table (Exam Answer)**

## **Definition**

Address translation is the process of converting a **virtual address generated by CPU** into a **physical address** using the page table.

---

## **Basic Concept**

- Virtual address = **VPN + Offset**
- Physical address = **PPN + Offset**
- Only **VPN is translated**, offset remains same

---

## **Steps in Address Translation**

1. CPU generates a **virtual address**
2. Split the address into:
    - **Virtual Page Number (VPN)**
    - **Page Offset**
3. Use VPN to **index the page table**
4. Read the corresponding **page table entry**
5. Check the **valid bit**:
    - If **1** → page present in memory
    - If **0** → **page fault occurs**
6. Obtain **Physical Page Number (PPN)**
7. Combine **PPN + Offset**  
    → Get **physical address**
8. Access data from **main memory**

 ----

## **Key Points (Write this 💯)**

- VPN is translated, **offset unchanged**
- Uses **page table in main memory**
- Valid bit determines **page fault**
- Final address = **PPN + Offset**
