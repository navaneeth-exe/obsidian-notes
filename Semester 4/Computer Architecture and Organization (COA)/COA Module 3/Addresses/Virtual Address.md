## **Definition**

A virtual address is the **address generated by the CPU** that refers to a location in the **virtual address space**, not directly to physical memory.

---

## **Basic Concept**

- Programs use **virtual addresses instead of physical addresses**
- These addresses are **translated into physical addresses** before accessing memory
- Helps in **efficient memory management**

---

## **Structure of Virtual Address**

A virtual address is divided into two parts:

1. **Virtual Page Number (VPN)**
    - Identifies the page
2. **Page Offset**
    - Identifies location within the page

👉 Virtual Address = **VPN + Offset**

---

## **Working**

- CPU generates virtual address
- Extract **VPN and offset**
- VPN is used to find corresponding physical page
- Offset remains unchanged
- Combined to form **physical address**

---

## **Example (Conceptual)**

- If page size = 4KB
- Offset = lower bits
- Remaining bits = page number

---

## **Key Points (Write this 💯)**

- Generated by CPU
- Divided into **VPN and offset**
- Needs **translation to physical address**
- Used in **virtual memory systems**
