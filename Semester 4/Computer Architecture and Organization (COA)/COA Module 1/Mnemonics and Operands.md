## 🔥 Mnemonics

👉 **Definition:**  
Mnemonics are **symbolic names used to represent operations** in assembly language.

### 💡 Examples:

- ADD → Addition
- SUB → Subtraction
- LW → Load word
- SW → Store word

👉 They make instructions **easy to read and remember**

---

## 🔥 Operands

👉 **Definition:**  
Operands specify the **data or location on which the operation is performed**

### 💡 Types:

- Register operands → data in CPU registers
- Memory operands → data in memory

---

## 🧠 Example

ADD s0, s1, s2

- **ADD** → mnemonic (operation)
- **s1, s2** → source operands
- **s0** → destination operand

👉 Meaning: add contents of s1 and s2, store result in s0

---

# ⚡ One-line Conclusion

👉 _Mnemonic = operation, Operand = data/location used in operation._
