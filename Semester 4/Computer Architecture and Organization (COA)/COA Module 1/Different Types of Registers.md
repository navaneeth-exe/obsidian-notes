### What are Registers?

Registers are **small, high-speed storage locations inside the CPU** used to temporarily hold **data, instructions, addresses, and intermediate results** during execution.

📌 **Key exam line:**

> Registers provide the fastest access to data for the CPU.

---

## Classification of Registers

Registers are broadly classified into:

1. **User-Visible Registers**
2. **Control and Status Registers**

---

## 1️⃣ User-Visible Registers

These registers are directly or indirectly accessible by the programmer.

### a) General Purpose Registers (GPR)

- Used to store **temporary data** and **intermediate results**.
- Can store operands during arithmetic and logical operations.
- Reduce memory access, improving performance.

📌 Example use: Loop counters, temporary variables.

---

### b) Accumulator (ACC)

- Stores **results of ALU operations**.
- Acts as an implicit operand for many instructions.
- Frequently used in arithmetic and logical operations.

📌 **Exam line:**

> Accumulator holds intermediate results produced by the ALU.

---

### c) Data Register (DR)

- Stores data fetched from memory.
- Holds data that is to be written back to memory.

📌 Works closely with MDR in many architectures.

---

### d) Address Register (AR)

- Stores memory addresses of data or instructions.
- Used to access specific memory locations.

---

## 2️⃣ Control and Status Registers

These registers control CPU operations and execution flow.

---

### a) Program Counter (PC) ⭐ VERY IMPORTANT

- Holds the **address of the next instruction** to be executed.
- Automatically incremented after each instruction fetch.
- Updated during branching and jumping.

📌 **Exam line:**

> Program Counter controls the sequence of instruction execution.

---

### b) Instruction Register (IR)

- Holds the **current instruction** being executed.
- Instruction is decoded from this register by the Control Unit.

📌 **Exam line:**

> Instruction Register stores the instruction currently under execution.

---

### c) Memory Address Register (MAR)

- Stores the **address of the memory location** to be accessed.
- Used during both read and write operations.

---

### d) Memory Data Register (MDR) / Memory Buffer Register (MBR)

- Stores **data being transferred** to or from memory.
- Acts as a buffer between CPU and memory.

📌 **Exam line:**

> MDR temporarily holds data read from or written to memory.

---

### e) Stack Pointer (SP)

- Holds the address of the **top of the stack**.
- Used during function calls, interrupts, and recursion.

📌 Stack = LIFO (Last In First Out)

---

### f) Status Register / Flag Register ⭐ IMPORTANT

- Stores condition codes generated by ALU.
- Common flags:
    - Zero flag (Z)
    - Carry flag (C)
    - Sign flag (S)
    - Overflow flag (V)

📌 **Exam line:**

> Status register indicates the outcome of ALU operations.

---

### g) Control Register

- Stores control information used by the CPU.
- Used to enable/disable interrupts and control execution modes.
