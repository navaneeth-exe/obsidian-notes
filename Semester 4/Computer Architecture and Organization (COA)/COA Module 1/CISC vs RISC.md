| **Aspect**                     | **CISC (Complex Instruction Set Computer)**      | **RISC (Reduced Instruction Set Computer)**        |
| ------------------------------ | ------------------------------------------------ | -------------------------------------------------- |
| **Basic Idea**                 | Uses complex instructions that do multiple tasks | Uses simple instructions, one task per instruction |
| **Instruction Set**            | Large and complex                                | Small and simple                                   |
| **Instruction Length**         | Variable                                         | Fixed                                              |
| **Execution Time**             | Takes multiple clock cycles                      | Usually 1 clock cycle                              |
| **Operations per Instruction** | Multiple operations in one instruction           | One operation per instruction                      |
| **Registers**                  | Few registers                                    | Many registers                                     |
| **Addressing Modes**           | Many                                             | Few                                                |
| **Hardware Design**            | Complex                                          | Simple                                             |
| **Control Unit**               | Microprogrammed                                  | Hardwired                                          |
| **Program Size**               | Smaller (fewer instructions needed)              | Larger (more instructions needed)                  |
| **Execution Speed**            | Slower                                           | Faster                                             |
| **Pipelining**                 | Difficult                                        | Easy and efficient                                 |
| **Power Consumption**          | Higher                                           | Lower                                              |
| **Compiler Role**              | Less important                                   | Very important (optimizes instructions)            |
| **Programming**                | Easier for programmers                           | Compiler does more work                            |
| **Cost**                       | Higher due to complex design                     | Lower due to simple design                         |
| **Examples**                   | Intel x86, AMD                                   | ARM, RISC-V, MIPS                                  |
| **Main Goal**                  | Reduce number of instructions                    | Increase execution speed                           |
| **Best For**                   | Desktop computers, complex tasks                 | Mobile devices, embedded systems                   |


# **CISC vs RISC — Sentence Form for Exams**

### **Basic Idea**

- CISC uses complex instructions where one instruction can perform multiple tasks.
- RISC uses simple instructions where each instruction performs only one task.

---

### **Instruction Set**

- CISC has a large and complex instruction set.
- RISC has a small and simple instruction set.

---

### **Instruction Length**

- CISC instructions have variable length.
- RISC instructions have fixed length.

---

### **Execution Time**

- CISC instructions usually take multiple clock cycles to execute.
- RISC instructions usually execute in one clock cycle.

---

### **Registers**

- CISC architecture uses fewer registers.
- RISC architecture uses a large number of registers.

---

### **Addressing Modes**

- CISC supports many addressing modes.
- RISC supports only a few addressing modes.

---

### **Hardware Design**

- CISC has a complex hardware design.
- RISC has a simple hardware design.

---

### **Control Unit**

- CISC uses a microprogrammed control unit.
- RISC uses a hardwired control unit.

---

### **Program Size**

- CISC programs are smaller because one instruction does more work.
- RISC programs are larger because more instructions are needed.

---

### **Execution Speed**

- CISC execution speed is slower.
- RISC execution speed is faster.

---

### **Pipelining**

- Pipelining is difficult in CISC.
- Pipelining is easy and efficient in RISC.

---

### **Power Consumption**

- CISC consumes more power.
- RISC consumes less power.

---

### **Examples**

- Examples of CISC processors are Intel x86 and AMD.
- Examples of RISC processors are ARM, RISC-V, and MIPS.
