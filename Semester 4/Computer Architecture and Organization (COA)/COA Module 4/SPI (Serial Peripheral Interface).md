## **Definition**

SPI is a **synchronous serial communication protocol** used for transferring data between a **master device (microcontroller)** and one or more **slave devices** at high speed.

---

## **Basic Concept**

- Communication is controlled by a **master**
- Data transfer is **synchronized using a clock signal**
- Supports **full-duplex communication**
- Used in embedded systems for fast peripheral interfacing

---

## **SPI Architecture**

An SPI system consists of:

1. **Master Controller**
    - Generates clock signal
    - Initiates and controls communication
2. **Slave Devices**
    - Receive data from master
    - Send response if required

---

## **SPI Communication Lines (VERY IMPORTANT 🔥)**

SPI uses **4 hardware signals**:

### **1. SCK (Serial Clock)**

- Generated by master
- Synchronizes data transfer

### **2. MOSI / SDO (Master Out Slave In)**

- Data sent from master → slave

### **3. MISO / SDI (Master In Slave Out)**

- Data sent from slave → master

### **4. SS / CE (Slave Select / Chip Enable)**

- Active LOW signal
- Selects the specific slave device

## **Working of SPI (Step-by-Step)**

1. Master selects slave by making **SS/CE = LOW**
2. Master generates **clock signal (SCK)**
3. Data is loaded into shift register
4. Data is transmitted **bit-by-bit via MOSI**
5. Slave reads data **on each clock pulse**
6. (Optional) Slave sends data back via MISO
7. After transmission, **SS is made HIGH** to end communication

---

## **Key Concept (Important Line 💡)**

- Data transmission happens **one bit per clock pulse**
- Transmission is **continuous and synchronized**

---

## **Advantages of SPI**

- Supports **full-duplex communication**
- **High-speed data transfer**
- No need for slave to generate clock
- Simple hardware design

---

## **Disadvantages of SPI**

- Requires **more wires (4 lines)**
- No built-in error checking
- Needs separate **SS line for each slave**

# **Communication Between Microcontroller and DAC using SPI**

## **Introduction**

In embedded systems, a **microcontroller communicates with a DAC (Digital-to-Analog Converter)** using the **SPI protocol** to convert digital data into corresponding analog output.

- Microcontroller acts as **Master**
- DAC acts as **Slave**
- Communication is **synchronous using clock**

## **SPI Signals Used**

1. **SCK (Serial Clock)**
    - Generated by microcontroller
    - Synchronizes data transfer
2. **MOSI (Master Out Slave In)**
    - Sends digital data to DAC
3. **MISO (Master In Slave Out)**
    - Usually not required for DAC
4. **CS/CE (Chip Select)**
    - Activates DAC (active LOW)

---

## **Working (Step-by-Step Explanation 🔥)**

1. Microcontroller selects DAC by making **CS/CE = LOW**  
    → DAC is activated
2. Microcontroller loads **digital data (e.g., 8/16-bit value)** into its register
3. Master generates **clock pulses (SCK)**  
    → Ensures synchronization
4. Data is transmitted **bit-by-bit through MOSI**  
    → DAC receives data through SDI
5. For **each clock pulse, one bit is transferred**
6. After complete transmission, **CS/CE is set HIGH**  
    → Ends communication
7. DAC processes received digital data  
    → Produces corresponding **analog voltage output**

---

## **Important Concept (Exam Line 💯)**

- Data is transferred **serially (one bit at a time)**
- Transfer is **synchronized with clock pulses**
- DAC updates output **only after full data is received**

---

## **Example (Optional for Extra Marks ⭐)**

- If microcontroller sends binary value **10110010**
- DAC converts it into proportional analog voltage based on reference
