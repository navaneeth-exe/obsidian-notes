## **Definition**

Address translation is the process of converting a **virtual (logical) address** generated by the CPU into a **physical address** in main memory.

---

## **Concept**

- CPU generates **virtual address (VA)**
- OS uses **Base Register + Limit Register**
- These ensure **memory protection + correct mapping**

---

## **Registers Used**

### **1. Base Register (Relocation Register)**

- Stores **starting physical address** of the process

### **2. Limit Register**

- Stores **size (range) of the process memory**

---

## **Working (Step-by-Step)**

![[Pasted image 20260412022504.png|520]]

### Steps:

1. CPU generates a **Virtual Address (VA)**
2. Check:  
    👉 If `VA < Limit Register` → valid  
    👉 Else → **Trap (error: illegal access)** 🚫
3. If valid:
```
     Physical Address (PA) = Base Register + VA
```
1. CPU accesses memory using PA

---

## **Formula**

$$
PA=Base+VA
$$

---

## **Example**

- Base = 1000
- Limit = 500
- VA = 200

👉 Check: 200 < 500 ✅  
👉 PA = 1000 + 200 = **1200**

---

## **Key Features**

- Provides **memory protection**
- Prevents process from accessing others’ memory
- Simple and fast method

---

## **Advantages**

- Easy to implement
- Low overhead
- Good for **contiguous memory allocation**

---

## **Disadvantages**

- Doesn’t support non-contiguous memory
- Can lead to **fragmentation**

---

## **Important Points (Write for Marks)**

- Virtual address is checked using **limit register**
- Base register is added to get **physical address**
- Invalid access → **trap to OS**
- Used in early OS memory management

---

## **One-Line Exam Killer**

👉 _Physical address is obtained by adding the base register to a valid virtual address within the limit._
