
## Definition

The **Burrows–Wheeler Transform** is a **reversible transformation** that rearranges the characters of a block of text.  
It **does not compress the data by itself**, but it **clusters similar characters together**, which makes the text easier for compression algorithms such as Move-To-Front and Huffman coding to compress.

---

## How the Forward Transform Works

The transformation of a string is done in three main steps:

### Step 1: Add Sentinel and Generate Rotations

A **special character $ ** is added to mark the end of the string.  
Then all possible **cyclic shifts (rotations)** of the string are generated by repeatedly moving the first character to the end.

|Rotations|
|---|
|BANANA$|
|ANANA$B|
|NANA$BA|
|ANA$BAN|
|NA$BANA|
|A$BANAN|
|$BANANA|
### Step 2: Sort the Rotations

All generated rotations are **sorted alphabetically (lexicographically)**.  
The special character `$` is treated as the **smallest character in the alphabet**.

|Sorted Rotations|
|---|
|$BANANA|
|A$BANAN|
|ANA$BAN|
|ANANA$B|
|BANANA$|
|NA$BANA|
|NANA$BA|
### Step 3: Extract the Last Column

After sorting, the **last character from each row** of the sorted rotations is taken.  
These characters together form the **Burrows–Wheeler Transform output**.

|Sorted Rotations|Last Column|
|---|---|
|$BANANA|A|
|A$BANAN|N|
|ANA$BAN|N|
|ANANA$B|B|
|BANANA$|$|
|NA$BANA|A|
|NANA$BA|A|

So,
```
BWT(BANANA$) = ANNB$AA
```

---
## How to Reverse the Transform (Inverse BWT)

The **Burrows–Wheeler Transform** is reversible, meaning the **original string can be reconstructed using only the last column of the BWT output**.

This reconstruction works using a rule called **LF Mapping (Last–First Mapping).**

### Magic Rule (LF Mapping)

The **i-th occurrence of a character in the Last column (L)** corresponds to the **i-th occurrence of the same character in the First column (F)**.

Example:

- First `a` in **L** corresponds to first `a` in **F**
    
- Second `a` in **L** corresponds to second `a` in **F**
    

---

## Steps to Reverse the BWT

### Step 1: Find the First Column (F)

Sort the **BWT output string alphabetically**.  
This sorted string forms the **first column (F)**.

Example
BWT Output (L):
```
ANNB$AA
```

Sorted → First Column (F):
```
$AAABNN
```


---

### Step 2: Match Character Ranks

Assign ranks to repeated characters to distinguish them.

Example

| L column | F column |
| -------- | -------- |
| A₁       | $        |
| N₁       | A₁       |
| N₂       | A₂       |
| B₁       | A₃       |
| $        | B₁       |
| A₂       | N₁       |
| A₃       | N₂       |

This helps map **each occurrence correctly**.

---

### Step 3: Follow the Trail

1. Start from the row containing the **sentinel symbol $**.
    
2. Look at the character in the **Last column (L)** of that row.
    
3. Jump to the row where that same ranked character appears in the **First column (F)**.
    
4. Repeat the process until the entire string is reconstructed.
    

This gradually rebuilds the original string **backwards**.

Reconstructed string: 
```
BANANA$
```

Original string:
```
BANANA
```

---

## Why BWT Works (Clustering Effect)

BWT works because characters that appear in similar contexts become **grouped together**.

Example in English:

- The sequence **“he”** often appears after **“t”** → “the”, “then”, “there”.
    

When rotations are sorted, all sequences starting with **“he”** come together, causing many **“t”** characters to appear consecutively in the last column.

This creates **long runs of the same character**, which are easy to compress using:

- Run-Length Encoding
    
- Move-To-Front coding
    
- Huffman coding
    

---

## Applications

- File compression tools like **bzip2**
    
- Bioinformatics for DNA sequence alignment using FM-index.
